# 4. Virtual Production (Generation)

## 4.1. 仮想生成のコンセプト

### "No Entity, No Tick"
既存の生成Mod（Botany Pots, Hnl/DML）が重い理由は、TileEntityの毎Tick処理やEntity生成にある。
本Modは、それらを**「Wasm内での純粋な数値計算」**に置き換える。

*   **Latency vs Throughput:** 生成速度を上げるのではなく、「計算をサボる（Lazy Evaluation）」ことで大量設置を許容する。
*   **Time as Cost:** 「時間がかかる」＝「その間サーバー負荷ゼロ」。

---

## 4.2. Mob牧場 (Hnl / Woot Hybrid)

### Hnl (Hostile Neural Networks) 方式
*   **特徴:** ユーザーがJSONで定義した固定リストからドロップ品を選ぶ。
*   **メリット:** 計算負荷が極小。
*   **デメリット:** Mod追加Mobへの対応が手動。

### Woot 方式
*   **特徴:** MobのLootTableを直接実行する。
*   **メリット:** 全Modに自動対応。
*   **デメリット:** LootContext生成コストがかかる。

### 本Modのアプローチ: "Simulation & Cache"
1.  **解析フェーズ (Java):**
    *   初回のみ、対象MobのLootTableを10,000回シミュレーションする。
    *   ドロップ品と確率の統計データを作成し、Wasmに送信する。
2.  **コスト化 (Wasm):**
    *   確率をエネルギーコストに変換する（期待値保存の法則）。
    *   例: ドロップ率 1% -> 基本コストの100倍のエネルギーで確定生産。
3.  **生産フェーズ:**
    *   Wasmはエネルギーを消費し、確定でアイテムを排出する。ランダム計算は行わない。

### コスト自動計算ロジック
*   **基本:** HP × 攻撃力 + 経験値。
*   **補正:** ドロップ品の資産価値（逆算査定）。弱いMobがレアアイテムを落とす場合のコスト崩壊を防ぐ。

## 4.3. 農業・鉱業・その他

### Botany Pots 方式 (農業)
*   **データ駆動:** JSONで「成長時間」と「ドロップ品」を定義。
*   **LootTable:** 起動時に解析してIDリスト化する。
*   **処理:** Wasm内タイマー管理のみ。

### Void Mining (虚空採掘)
*   **バイオーム依存:** スキャナーで登録したバイオームに基づき、重み付けリストから鉱石を抽出。
*   **メリット:** クァーリー（露天掘り）によるチャンク読み込み負荷を回避。

### Loot Chest (ダンジョンチェスト)
*   **スキャナー:** 現地でチェストをスキャンし、LootTable ID (`minecraft:chests/simple_dungeon` 等) と座標・バイオームを記録。
*   **消耗品:** データチップに耐久値を持たせ、完全放置によるバランス崩壊を防ぐ（再スキャンの動機付け）。
*   **Luck:** 解析時は Luck=0 で計算。アップグレードパーツでLuck補正を可能にする。

### 村人・ピグリン取引
*   **ピグリン:** LootTable解析による確定交換（自販機化）。
*   **村人:** 捕獲時に取引リスト（MerchantOffers）をスナップショットとして保存。Entityは消去し、データとして取引を行う。

## 4.4. プロジェクト構成: Core vs Production

### Suite構成
*   **HyperVisor-Core:** 倉庫、ネットワーク、Wasmランタイム。
*   **HyperVisor-Production:** 生成機能。Coreに依存するアドオン。

### Wasm Direct Bridge
*   生産機（Production）と倉庫（Core）は、物理パイプを使わず**「Wasmメモリ間での数値渡し」**で直結する。
*   **オーバーヘッド:** Javaブリッジ経由の呼び出しコストは、バニラのItemStack移動に比べれば無視できるレベル。
*   **メモリ効率:** Productionを使わない場合、その分のWasmメモリは確保されない。

### バランス調整
*   **Config:** `enable_loot_gen = false` のように、機能単位で無効化可能にする。
*   **Catalyst:** 強力な生成には、エネルギーだけでなく「触媒（現地のアイテム）」を要求する。
