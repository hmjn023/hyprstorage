# 3. AE2 Integration & Crafting

## 3.1. AE2 (Applied Energistics 2) との関係性

### なぜ連携するのか？
*   **強み:** AE2は複雑なクラフト、NBT処理、アドオン（Wireless Terminal等）のエコシステムが完成している。
*   **弱み:** 再帰計算によるラグ、GC負荷、大規模ネットワークでのTPS低下。
*   **戦略:** AE2の優れたUI/UXを利用しつつ、重い計算部分をWasmにオフロードする。

### 連携モデル: "Parasite Controller" (寄生型)
本Modは「AE2のネットワークの一部」として振る舞い、内部データを吸い上げる。

1.  **Data Heist (データ強奪):**
    *   本ModのブロックをAE2ネットワークに接続。
    *   ME Drive内のセル等からアイテムを吸い出し、Wasmメモリへ移動させる。
2.  **Virtual Provider (仮想ストレージ):**
    *   AE2に対して `IMEInventory` (外部ストレージ) として振る舞う。
    *   AE2側からは「無限の容量を持つドライブ」に見える。
3.  **メリット:**
    *   ユーザーは既存のAE2拠点を壊さずに導入できる。
    *   在庫管理の計算負荷がWasmに移るため、サーバーが軽くなる。

## 3.2. 階層型クラフトシステム (Hierarchical Crafting)

「単純作業」と「複雑作業」を分離し、最適化する。

### Tier 1: Virtual Assembly (Wasm内部)
*   **対象:** 単純なレシピ（1:1変換、粉砕、圧縮）、中間素材（鉄板、回路、ワイヤー）。
*   **処理:**
    *   物理的なアイテム移動や機械の動作を行わない。
    *   Wasmメモリ上で `Input ID` の数値を減らし、`Output ID` の数値を増やすだけ。
*   **速度:** 1 Tickに数百万回のクラフトが可能。

### Tier 2: Orchestration Mode (AE2スレーブ)
*   **対象:** 複雑なNBT操作、他Modの機械（Mekanismの化学反応など）、AE2アドオン依存レシピ。
*   **役割:** 本Modは「監督」。AE2は「下請け工場」。
*   **Micro-Orders (マイクロ発注):**
    *   巨大な注文（例: ソーラーパネル1000個）をそのままAE2に投げない（再帰計算ラグ防止）。
    *   Wasmが中間素材の在庫を監視し、不足分を小口でAE2の `ICraftingGrid` に発注する。
*   **Promise / Token:**
    *   外部機械に材料を投入後、Wasmは「完了トークン」を発行し、機械を忘れる（ポーリングしない）。
    *   製品が戻ってきた時点でトークンを消し込み、タスク完了とする。

## 3.3. エンチャント・NBTの処理

Wasmは不定形データ（NBT）が苦手なため、特殊な処理を行う。

### ID化とフラット化
*   **Java側:** エンチャントの組み合わせごとにユニークIDを発行する。
    *   `{id:"sharpness", lvl:5}` -> `ID: 9901`
*   **Wasm側:** 単なる整数 `ID: 9901` として在庫管理する。

### 生産時のマトリックス計算
*   **合成 (Anvil):** Rust内で「IDとレベルの配列」として計算し、合成結果のIDを返す。
*   **選別:** 「鋭さV以上」などの条件を数値比較で高速にフィルタリングする。

### 未鑑定品アプローチ (Loot Generation)
*   ダンジョンチェスト等の生成時、Wasm内では「エンチャントルールID付きの素体」として管理する。
*   Java側に排出される瞬間に、ルールに基づいてランダムエンチャントを付与する（組み合わせ爆発の防止）。
